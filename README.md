# 医療DX シリーズ — FHIR / JLAC11 実装ガイド

臨床検査データを HL7 FHIR R4（JP Core / JP-CLINS）と JLAC11 で**相互運用可能にする**実装ガイド。「FHIR 対応は要件」で止まっていた話の How 版。

## 公開ページ
➡ **https://kioh-daiv.github.io/lis-fhir-jlac-implementation/**

## 設計思想
- **横の証明（網羅性）**：① 構造（FHIR の 3 リソース：DiagnosticReport/Observation/Specimen）× ② 意味（コード体系：JLAC11・UCUM）の 2 軸で、相互運用の両輪を漏れなく押さえる。対象/対象外を明示。
- **縦の証明**：標準の地図 → リソース設計 → コード変換 → 共有サービス接続 → バリデーション、と実装の流れで分解。残リスクは 4 分類で引き取り。

## 構成
- スコープと網羅性の根拠
- §1 標準の地図（FHIR R4 / JP Core / JP-CLINS / JLAC11 / UCUM）
- §2 検査を表す 3 つの FHIR リソース
- §3 コード変換（ローカル→JLAC11、HL7 v2→FHIR）
- §4 電子カルテ情報共有サービスへの接続（JP-CLINS、2文書5情報＋患者サマリー）
- §5 バリデーションと落とし穴
- 出典（一次資料・公的情報）

## 位置づけ
深掘り編。②が「標準対応はなぜ要件か（Why）」、本編が「どう実装するか（How）」、移行プレイブックが「いつ変換を仕込むか（When）」、データインテグリティ編が「変換しても真正性を保つ」に接続。

## 注意
2026 年 5 月時点。FHIR R4 / JP Core（1.3.0-dev 開発中）/ JP-CLINS / JLAC11 は版が更新される。各実装ガイドの最新版・コード表と自院システム・ベンダー仕様で検証すること。
